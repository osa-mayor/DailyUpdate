## 1. [Crafter: A Multi-Agent Harness for Editable Scientific Figure Generation from Diverse Inputs](https://huggingface.co/papers/2605.30611)

**arXiv**: https://arxiv.org/abs/2605.30611

**태그**: Multi-Agent, SVG-Generation, Scientific-Visualization, Automation, Agent, Benchmark

![Thumbnail](images/Crafter_A_Multi-Agent_Harness_for_Editable_Scientific_Figure_Generation_from_Diverse_Inputs_img.jpg)

### 📌 한 줄 요약
다양한 입력 조건과 피규어 유형에 대응하며 편집 가능한 SVG 출력을 생성하는 멀티 에이전트 기반 과학 도식 생성 프레임워크

### 🔑 핵심 포인트
- 다양한 도식 유형과 입력 조건에 범용적으로 대응하는 멀티 에이전트 하네스(Crafter) 제안
- 래스터 출력을 편집 가능한 SVG로 변환하여 사후 수정이 가능하게 하는 CraftEditor 개발
- 인간 수준의 품질 주석이 포함된 새로운 벤치마크 CraftBench 구축

### 🧑‍💻 개발자 관점
단순 이미지 생성을 넘어, 구조화된 데이터(SVG)를 생성함으로써 생성형 AI 결과물의 고질적인 문제인 '수정 불가능성'을 해결하는 실무적 접근법을 제시합니다.

### 🚀 실무 적용 아이디어
- 제공된 GitHub 코드를 통해 다양한 입력 조건(텍inal, data-driven 등)에서의 생성 품질 테스트
- CraftEditor를 활용하여 생성된 SVG의 레이어 구조와 편집 용이성 검증
- 기존 단일 모델 기반 생성 방식과 에이전트 기반 방식의 워크플로우 효율성 비교

### ⚠️ 리스크/한계
- 에이전트 기반 구조로 인해 단일 모델 대비 추론 비용 및 시간이 높을 수 있음
- 복잡도가 매우 높은 도식의 경우 에이전트 간 협업 과정에서 논리적 오류가 발생할 가능성

### 📝 초록 기반 상세 설명
과학 논문 작성을 위한 고품질 도식 생성은 노동 집약적이며, 기존 자동화 시스템은 단일 유형에 국한되거나 수정이 불가능한 래스터 이미지만을 생성하는 한계가 있습니다. 본 논문은 도식이 이산적인 의미적 구성 요소로 이루어져 있다는 점에 착안하여, 아키텍처 변경 없이 다양한 유형에 대응하는 멀티 에이전트 하네스 구조를 제안합니다. 이를 위해 범용적인 도식 생성을 위한 'Crafter'와 생성된 이미지를 편집 가능한 SVG로 변환하는 'CraftEditor'를 개발했습니다. 또한 세 가지 도식 유형과 네 가지 입력 조건을 포함하는 벤치마크인 'CraftBench'를 도입했습니다. 실험 결과, Crafter는 기존 단일 모델 및 에이전트 베이스라인을 상회하는 성능을 보였으며, CraftEditor는 높은 충실도로 편집 가능한 결과물을 생성함을 입증했습니다.

## 2. [On the Scaling of PEFT: Towards Million Personal Models of Trillion Parameters](https://huggingface.co/papers/2606.02437)
**Upvotes**: 55 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.02437

**태그**: PEFT, Personalization, LLM-Ops, Scalability, Vision, Evaluation

![Thumbnail](images/On_the_Scaling_of_PEFT_Towards_Million_Personal_Models_of_Trillion_Parameters_img.jpg)

### 📌 한 줄 요약
PEFT를 단순 비용 절감 수단이 아닌, 거대 모델 위에 개인화된 상태를 유지하는 '개인용 모델(Personal Models)' 구축 프레임워크로 재정의함

### 🔑 핵심 포인트
- PEFT를 공유 지식(Base Model)과 개인화된 상태(Adapter)를 분리하는 아키텍처로 재정의
- 확장성 확보를 위한 세 가지 축(Scale Up, Down, Out) 제시
- 수많은 개인화된 어댑터를 관리하기 위한 인프라(MinT) 개념 도입

### 🧑‍💻 개발자 관점
거대 모델을 매번 새로 학습할 필요 없이, 아주 작은 어댑터만 교체함으로써 수만 명의 사용자에게 맞춤형 AI를 저비용으로 제공할 수 있는 설계 방향을 제시합니다.

### 🚀 실무 적용 아이디어
- 특정 도구 사용 습관이나 개인 말투를 학습시킨 작은 어댑터 생성 실험
- 하나의 베이스 모델에 여러 사용자 프로필 어댑터를 스위칭하며 성능 변화 관찰
- 어댑터 크기를 극단적으로 줄였을 때(Scale Down)의 성능 유지 한계 테스트

### ⚠️ 리스크/한계
- 어댑터 간의 간섭이나 데이터 오염 문제
- 수많은 어댑터를 실시간으로 서빙할 때의 인프라 복잡도 및 관리 비용

### 📝 초록 기반 상세 설명
기존의 PEFT는 주로 풀 파인튜닝의 저비용 대안으로만 취급되어 왔습니다. 본 논문은 PEFT를 강력한 공유 기반 모델 위에 개인의 선호도, 기술, 기억을 담는 지속적인 로컬 상태로 활용하는 새로운 관점을 제시합니다. 이를 위해 Scale Up(강력한 사전 학습), Scale Down(어댑터의 최소화), Scale Out(수많은 인스턴스 공존)이라는 세 가지 확장 축을 중심으로 문제를 구조화합니다. 또한, 어댑터의 식별, 버전 관리, 평가 및 서빙을 관리하기 위한 인프라 예시인 MinT를 제안합니다. 결과적으로 PEFT가 단순한 예산 절약 수단을 넘어, 수조 개의 파라미터를 가진 모델을 개인화하는 압축된 기질(substrate)이 될 수 있음을 시사합니다.

- 도구 시퀀스 기반의 역방향 태스크 생성 방식(Task Synthesis from Tool Sequence Evolution) 제안

- 클러스터링 및 반복적 난이도 진화 과정을 통한 자동화된 고난도 벤치마크 구축

에이전트 개발 시 기존 벤치마크 점수에 안주하지 않고, 실제 복잡한 도구 사용 환경에서의 성능을 검증할 수 있는 새로운 평가 기준을 제시합니다.

### 🚀 실무 적용 아이디어
- 현재 개발 중인 에이전트의 도구 사용 시퀀스를 추출하여 TASTE 방식의 자동 생성 파이프라인 적용 테스트

- 도구 조합의 다양성을 확보하기 위한 클러스터링 및 난이도 조절 로직 실험

### ⚠️ 리스크/한계
- 생성된 태스크가 특정 도구 조합에 편향될 가능성
- 자동 생성된 시나리오의 논리적 일관성 및 자연어 품질 유지 문제

### 📝 초록 기반 상세 설명
에이전트 능력이 발전함에 따라 기존 벤치마크가 쉽게 포화되는 문제가 발생하고 있으며, 새로운 태스크를 수동으로 구축하는 것은 비용과 시간이 많이 소요됩니다. 또한 기존 방식은 자연어 시나리오를 도구 시퀀스로 매핑하는 방식이라 도구 활용 패턴이 제한적이라는 한계가 있습니다. 본 논문은 태스크 생성 과정을 역전시켜 도구 시퀀스에서 태스크를 도출하는 TASTE 프레임워크를 제안합니다. TASTE는 LLM 피드백 기반의 적응형 대조 n-gram 모델을 통해 다양한 도구 조합을 포함하는 유효한 시퀀스를 샘플링하고, 클러스터링과 반복적인 난이도 진화를 통해 완성도 높은 벤치마크를 자동 생성합니다. 이를 통해 구축된 τ^c-Bench에서 기존 모델들은 성능 저하를 보였으며, 도구 조합의 다양성 또한 크게 향상되었습니다. 결과적으로 TASTE는 미래 에이전트의 능력을 지속적이고 확장 가능하게 평가할 수 있는 기반을 제공합니다.

### 📝 초록 기반 상세 설명

- 실행 전 후보 동작을 필터링하여 물리적 제약 위반을 방지하는 예측-제어 인터페이스 구축

### 🧑‍💻 개발자 관점
로봇 제어 모델이 생성한 경로가 실제 하드웨어에서 구동 가능한지(예: 관절 한계, 충돌 등)를 수학적으로 검증하는 가이드라인을 제공합니다.

### 🚀 실무 적용 아이디어
- 기존 학습된 로봇 정책(Policy)에 물리적 제약 조건 기반의 필터 레이어 추가 실험
- 단순 RMSE 기반의 모델 평가와 제안된 물리적 허용성 지표 간의 상관관계 분석
- 시뮬레이션 환경에서 제안된 게이트가 실제 하드웨어 손상을 얼마나 방지할 수 있는지 검증

### ⚠️ 리스크/한계
- 물리적 검증 과정이 추가됨에 따라 실시간 제어 루프에서의 연산 오버헤드 발생 가능성
- 정교한 물리 모델이 필요하며, 모델이 실제 물리 법칙과 다를 경우 오작동 위험

### 📝 초록 기반 상세 설명
물리적 AI 시스템이 생성하는 상태 예측이나 계획이 낮은 RMSE를 기록하더라도 실제 환경에서는 실행 불가능한 경우가 많습니다. 본 논문은 예측된 결과가 물리적 법칙을 준수하는지 검증하는 '물리적 허용성(Physical Admissibility)' 개념을 제안합니다. 이를 위해 디코딩된 제안을 후보 역학으로 간주하고 운동학적, 역학적, 지평선 조건 등을 통해 평가하는 예측-제어 인터페이스를 구축했습니다. 실험 결과, 단순 RMSE보다 제안된 게이트 방식이 유효하지 않은 제안을 식별하는 데 훨씬 높은 성능을 보였습니다. 최종적으로 이 필터링 기법은 작업 성공률을 유지하면서도 물리적으로 불가능한 동작을 효과적으로 차단합니다.

## 50. [ACL-Verbatim: hallucination-free question answering for research](https://huggingface.co/papers/2605.21102)
**Upvotes**: 2 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2605.21102

**태그**: RAG, Information Extraction, NLP, Benchmark, Vision, Evaluation

![Thumbnail](images/ACL-Verbatim_hallucination-free_question_answering_for_research_img.jpg)

### 📌 한 줄 요약
연구 논문 데이터에서 환각(Hallucination) 없이 정확한 답변을 추출하기 위한 텍스트 스팬 매핑 벤치마크 및 모델 제안

### 🔑 핵심 포인트
- 연구 논문 특화 텍스트 스팬 매핑(Extractive QA) 벤치마크 데이터셋 구축
- ScIRGen 기반의 합성 질의 생성 파이프라인을 통한 고품질 데이터 확보
- LLM보다 효율적이고 정확한 소형 모델(ModernBERT) 기반의 추출 성능 입증

### 🧑‍💻 개발자 관점
RAG 시스템 구축 시 생성형 모델의 환각을 방지하기 위해, 답변을 생성하는 대신 원문 텍스트를 직접 추출하는 방식의 유효성을 확인해 줍니다.

### 🚀 실무 적용 아이디어
- ScIRGen 방법론을 활용한 도메인 특화 합성 데이터 생성 파이프라인 테스트
- LLM 기반 생성형 RAG와 소형 모델 기반 추출형 RAG의 성능 및 비용 비교 실험
- 제공된 벤치마크 데이터셋을 활용한 기존 추출 모델의 성능 검증

### ⚠️ 리스크/한계
- 합성 데이터(Synthetic data) 기반 학습으로 인한 데이터 편향 가능성
- 특정 도메인(학술 논문)에 최적화된 결과로 일반적인 대화형 태스크로의 확장성 제한

### 📝 초록 기반 상세 설명
학술 연구를 위한 AI 도구는 신뢰할 수 있는 정보 수집이 필수적이지만, LLM의 환각 현상으로 인해 정확한 정보 추출에 어려움이 있습니다. 본 연구는 ACL Anthology 논문을 대상으로 사용자 질의를 문서 내 실제 텍스트 구간과 직접 매핑하는 VerbatimRAG 시스템을 적용했습니다. 이를 위해 연구 논문 내 질의-텍스트 구간 매핑을 위한 새로운 Ground Truth 데이터셋을 구축했습니다. 데이터셋 구축에는 ScIRGen 방법론을 활용한 합성 질의 생성 파이프라인이 사용되었습니다. 실험 결과, 150M 파라미터 규모의 ModernBERT 모델이 강력한 LLM 추출기보다 높은 성능을 기록하며 정답 구간 추출 능력을 입증했습니다.
