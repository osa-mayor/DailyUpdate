**arXiv**: https://arxiv.org/abs/2606.12397

**태그**: MoE, Router Design, Optimization, Deep Learning, Safety

![Thumbnail](images/Redesign_Mixture-of-Experts_Routers_with_Manifold_Power_Iteration_img.jpg)

### 📌 한 줄 요약
라우터 행렬을 전문가(Expert) 행렬의 주성분 방향과 일치시켜 MoE 모델의 전문가 선택 효율성을 극대화하는 기법

### 🔑 핵심 포인트
- 라우터 행을 전문가 행렬의 주성분 방향(Principal Singular Direction)에 정렬하는 새로운 설계 원칙 제안
- 효율적이고 안정적인 수렴을 위한 'Power-then-Retract' 기반의 Manifold Power Iteration(MPI) 알고리즘 개발
- 다양한 파라미터 규모(1B~11B)에서의 실험을 통한 MoE 모델 성능 최적화 입증

### 🧑‍💻 개발자 관점
MoE 모델 설계 시 라우터와 전문가 간의 수학적 정렬을 통해 토큰-전문가 매칭의 정확도를 높일 수 있는 방법론을 제시합니다.

### 🚀 실무 적용 아이디어
- 기존 MoE 모델의 라우터 가중치와 전문가 가중치 간의 코사인 유사도 분석
- MPI 알고리즘을 기존 라우터 학습 루프에 통합하여 수렴 속도 및 성능 변화 관찰
- 다양한 전문가 행렬 크기(Expert dimension)에 따른 MPI의 안정성 테스트

### ⚠️ 리스크/한계
- 학습 과정에서 추가적인 연산 비용(Power iteration)이 발생할 수 있음
- 특정 규모 이상의 모델에서 수렴 안정성이 유지되는지에 대한 추가 검증 필요

### 📝 초록 기반 상세 설명
MoE 모델에서 라우터는 토큰과 전문가 간의 유사도를 계산하여 활성화할 전문가를 결정하는 핵심 역할을 합니다. 하지만 기존 라우터 설계에는 전문가 행렬의 정보를 효과적으로 압축하여 표현할 수 있는 명확한 원칙이 부족했습니다. 본 논문은 라우터 행렬의 각 행을 해당 전문가 행렬의 주 특이 방향(Principal Singular Direction)과 정렬하는 방식을 제안합니다. 이를 위해 'Power-then-Retract' 패러다임을 기반으로 한 Manifold Power Iteration(MPI) 알고리즘을 도입하여 효율성과 안정성을 동시에 확보했습니다. 이론적으로 MPI는 라우터 행을 전문가의 주성분 방향으로 수렴시킴을 증명했습니다. 1B에서 11B 규모의 다양한 MoE 모델 사전 학습 실험을 통해 이 정렬 방식이 모델 성능 향상에 기여함을 확인했습니다.

- 단일 루프가 아닌, 전략(Planner)과 실행(Worker) 계층을 분리한 멀티 에이전트 환경 구축하기
- 실험 실패 사례를 '부정적 증거'로 데이터베이스화하여 재사용하는 워크플로우 테스트하기

### ⚠️ 리스크/한계
- 트리 구조가 복잡해질수록 관리 비용 및 토큰 소모량 급증 가능성
- 잘못된 통찰(False Insight)이 트리 전체로 전파될 경우 발생하는 오류 누적 리스크

HTR는 가설, 결과물, 증거, 통찰을 트리 구조로 연결하여 시간이 지나도 재사용 가능한 지식을 유지합니다. 실험 결과, Arbor는 모델 학습 및 데이터 합성 등 다양한 자율 최적화(AO) 작업에서 기존 모델 대비 월등한 성능 향상을 달성했습니다. 특히 MLE-Bench Lite에서 강력한 성능을 입증하며 자율 연구 에이전트의 가능성을 보여주었습니다.

**태그**: Reward Modeling, VLM, Knowledge Distillation, Text-to-Image, Reasoning, Vision, Evaluation, Inference, Distillation

![Thumbnail](images/Beyond_Scalar_Rewards_by_Internalizing_Reasoning_into_Score_Distributions_img.jpg)

### 📌 한 줄 요약
추론 과정을 점수 분포에 내재화하여, 높은 성능의 추론형 보상 모델을 효율적인 경량 모델로 증류하는 Z-Reward 프레임워크 제안

### 🔑 핵심 포인트
- GDSO(Group-wise Direct Score Optimization): 분포 기대값과 점수 차이를 결합하여 추론 기반 점수 분포를 학습하는 방법론
- RISD(Reasoning-Internalized Score Distillation): 추론 체인 없이도 Teacher의 추론 능력이 반영된 점수 분포를 Student 모델로 전이하는 기술
- Teacher-Student 구조를 통한 추론 성능과 배포 효율성 간의 트레이드오프 해결

### 🧑‍💻 개발자 관점
고비용의 추론(Reasoning) 과정을 모델의 가중치(점수 분포)에 내재화함으로써, 추론 시에는 빠른 속도를 유지하면서도 정교한 보상 신호를 얻을 수 있습니다.

### 🚀 실무 적용 아이디어
