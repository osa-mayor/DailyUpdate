![Thumbnail](images/Code2LoRA_Hypernetwork-Generated_Adapters_for_Code_Language_Models_under_Software_Evolution_img.jpg)

### 📌 한 줄 요약
하이퍼네트워크를 통해 코드 저장소별 맞춤형 LoRA 어댑터를 생성하여, 추론 비용 증가 없이 프로젝트 컨텍스트를 주입하는 프레임워크

### 🔑 핵심 포인트
- 하이퍼네트워크를 이용한 저장소별 맞춤형 LoRA 어댑터 자동 생성
- 추론 시 추가적인 토큰 오버헤드 없이 프로젝트 컨텍스트 주입 가능
- 정적 코드베이스(Static)와 지속적 업데이트(Evo) 시나리오 모두 지원

### 🧑‍💻 개발자 관점
대규모 코드 저장소의 컨텍스트를 RAG처럼 길게 넣지 않고도, 가벼운 어댑터 교체만으로 프로젝트 특화 모델을 운영할 수 있습니다.

### 🚀 실무 적용 아이디어
- 자사 코드 저장소의 스냅샷을 활용한 Code2LoRA-Static 성능 테스트
- CI/CD 파이프라인에 코드 변경 시 어댑터를 업데이트하는 Evo 방식 적용 실험
- 기존 RAG 방식 대비 추론 속도 및 비용 절감 효과 비교

### ⚠️ 리스크/한계
- 하이퍼네트워크 학습을 위한 초기 데이터 준비 및 학습 비용 발생
- 매우 빈번한 코드 변경 시 GRU 기반 상태 업데이트의 안정성 검증 필요

### 📝 초록 기반 상세 설명
코드 언어 모델이 프로젝트의 API 및 컨벤션을 이해하기 위해 긴 컨텍스트를 입력하거나 매번 파인튜닝하는 방식은 비용이 높고 코드 변화에 취약합니다. 이를 해결하기 위해 하이퍼네트워크를 활용해 저장소별 LoRA 어댑터를 생성하는 Code2LoRA를 제안합니다. 이 프레임워크는 정적인 코드베이스를 위한 Static 방식과 코드 변경 사항을 반영하는 Evo 방식을 모두 지원합니다. 실험 결과, 제안 방식은 기존 LoRA 수준의 성능을 유지하면서도 추론 시 토큰 오버헤드 없이 효율적인 컨텍스트 주입이 가능함을 입증했습니다. 특히 진화하는 코드베이스 환경에서 기존 방식보다 우수한 성능을 보였습니다.

## 2. [ArcANE: Do Role-Playing Language Agents Stay in Character at the Right Time?](https://huggingface.co/papers/2606.05553)
**Upvotes**: 45 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.05553

**태그**: Agent, Persona, Benchmark, NLP, RAG, Evaluation

![Thumbnail](images/ArcANE_Do_Role-Playing_Language_Agents_Stay_in_Character_at_the_Right_Time_img.jpg)

### 📌 한 줄 요약
캐릭터의 심리적 변화(Character Arc)를 반영하여 페르소나의 일관성을 평가하는 새로운 벤치마크 ArcANE 제안

### 🔑 핵심 포인트
- 심리적 궤적(Character Arc)을 반영한 동적 페르소나 평가 프레임워크 구축
- 원작 텍스트 내 상황뿐만 아니라 원작 외(Out-of-distribution) 상황에서의 캐릭터 일관성 검증
- 캐릭터 아크 정보를 컨텍스트로 활용할 때의 성능 우위 입증 및 전용 모델(ArcANE-8B/32B) 개발

### 🧑‍💻 개발자 관점
단순한 정보 검색(RAG)을 넘어, 시간에 따라 변화하는 복잡한 페르소나를 가진 에이전트를 설계하고 평가할 때 필수적인 방법론을 제공합니다.

### 🚀 실무 적용 아이디어
- 에이전트 설계 시 고정된 시스템 프롬프트 대신 캐릭터의 성장/변화 단계를 포함한 동적 컨텍스트 주입 실험
- LLM이 특정 상황에서 캐릭터의 심리적 변화를 올바르게 추론하는지 확인하기 위한 시나리오 테스트
- 제공된 ArcANE 데이터셋을 활용하여 에이전트의 페르소나 유지 능력 벤치마킹

### ⚠️ 리스크/한계
- 자동 구축된 벤치마크이므로 인간의 주관적 해석과 차이가 있을 수 있음
- 심리적 변화가 극단적인 경우 모델이 일관성을 유지하기 어려울 수 있음

### 📝 초록 기반 상세 설명
기존의 역할 수행 언어 에이전트(RPLA) 평가 방식은 고정된 페르소나 유지에만 집중하여, 서사 진행에 따른 캐릭터의 심리적 변화를 측정하지 못하는 한계가 있습니다. 이를 해결하기 위해 17권의 소설과 80명의 주요 캐릭터를 기반으로 한 자동 구축 벤치마크인 ArcANE을 도입합니다. ArcANE은 서사를 심리적 축에 따라 여러 단계로 나누고, 각 단계에서 동일한 시나리오를 제시하여 캐릭터의 변화 양상을 평가합니다. 실험 결과, 캐릭터의 서사 흐름(Character Arc)을 컨텍스트로 제공하는 방식이 다른 전략보다 월등한 성능을 보였으며, 특히 원작에 없는 상황에서도 캐릭터의 일관성을 유지하는 데 효과적이었습니다. 또한, 해당 데이터로 미세 조정된 ArcANE-8B/32B 모델은 원작 외 시나리오에서 더욱 강력한 성능을 입증했습니다.

## 3. [TIDE: Proactive Multi-Problem Discovery via Template-Guided Iteration](https://huggingface.co/papers/2606.04743)
**Upvotes**: 38 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.04743

**태그**: AI Agent, Problem Discovery, Iterative Reasoning, Software Engineering, Agent, RAG, Distillation

![Thumbnail](images/TIDE_Proactive_Multi-Problem_Discovery_via_Template-Guided_Iteration_img.jpg)

### 📌 한 줄 요약
사용자의 명시적 요청 없이도 컨텍스트 내 숨겨진 다중 문제를 능동적으로 찾아내고 해결하는 반복적 에이전트 프레임워크

### 🔑 핵심 포인트
- 반복적 발견(Iterative Discovery): 이전 라운드 결과를 조건부로 활용하여 중복을 피하고 탐색 범위를 확장하는 방식
- 사고 템플릿(Thought Templates): 문제 유형을 규정하고 맥락적 신호를 연결하는 재사용 가능한 스키마 활용
- 능동적 문제 발굴: 사용자 요청에 의존하지 않고 컨텍스트 내 잠재적 이슈를 선제적으로 식별

### 🧑‍💻 개발자 관점
코드 리뷰나 워크스페이스 관리 시, 개발자가 미처 인지하지 못한 잠재적 버그나 설정 오류를 에이전트가 스스로 찾아내어 제안할 수 있습니다.

### 🚀 실무 적용 아이디어
- 기존 단일 프롬프트 기반 에이전트에 반복적 피드백 루프를 적용하여 문제 탐색 범위 변화 관찰
- 특정 도메인(예: 보안 취약점 탐지)에 특화된 사고 템플릿 설계 및 적용 실험
- 에이전트가 제안하는 문제의 중복성 및 커버리지 정량적 평가

### ⚠️ 리스크/한계
- 반복적 루프 수행 시 발생하는 추론 비용(Token usage) 및 지연 시간 증가
- 템플릿에 의존할 경우 정의되지 않은 새로운 유형의 문제에 대한 대응력 저하 가능성

### 📝 초록 기반 상세 설명
현재의 AI 에이전트는 사용자의 명시적 요청에만 반응하므로, 컨텍스트 내에 숨겨진 다양한 잠재적 문제들을 놓치는 한계가 있습니다. 본 논문은 컨텍스트에서 여러 개의 숨겨진 문제를 동시에 발견하고 근거를 제시하며 해결책을 제안하는 과제를 정의합니다. 이를 위해 제안된 TIDE 프레임워크는 두 가지 핵심 메커니즘을 사용합니다. 첫째, 이미 발견된 문제를 고려하여 새로운 후보를 확장하는 '반복적 발견(Iterative Discovery)' 방식을 통해 탐색 범위를 넓힙니다. 둘째, 과거 사례에서 추출된 '사고 템플릿(Thought Templates)'을 통해 특정 문제 클래스에 집중하고 예측의 정확도를 높입니다. 실험 결과, 개인 작업 공간 및 소프트웨어 저장소 환경에서 기존의 단일 패스 방식이나 병렬 멀티 에이전트 방식보다 높은 문제 커버리지와 해결 능력을 입증했습니다.

## 4. [AdaPlanBench: Evaluating Adaptive Planning in Large Language Model Agents under World and User Constraints](https://huggingface.co/papers/2606.05622)

### 📌 한 줄 요약
점진적으로 공개되는 환경 및 사용자 제약 조건 하에서 LLM 에이전트의 적응형 계획 수립 능력을 평가하는 동적 벤치마크 제안

### 🔑 핵심 포인트
- 점진적으로 공개되는 이중 제약(World & User) 하에서의 적응형 계획 수립 평가 프레임워크 개발
- 제약 조건 위반 시에만 피드백이 제공되는 동적 인터랙티브 프로토콜 설계
- 제약 조건 누적에 따른 에이전트의 성능 저하 및 사용자 제약 조건에 대한 취약성 규명

### 🧑‍💻 개발자 관점
에이전트가 불완전한 정보 속에서 피드백을 통해 계획을 수정해야 하는 실무적 시나리오(예: 개인 비서, 로봇 제어)를 위한 핵심 평가 지표를 제공합니다.

### 🚀 실무 적용 아이디어
- 에이전트의 피드백 수용 및 계획 수정 로직(Re-planning loop) 최적화 실험
- 제약 조건이 누적됨에 따라 발생하는 컨텍스트 손실 및 추론 오류 분석
- 사용자 의도(User constraint)를 더 정확히 파악하기 위한 프롬프트 엔지니어링 테스트

### ⚠️ 리스크/한계
- 가사 작업 중심의 시나리오로 인해 다른 도메인으로의 일반화 가능성 검증 필요
- 제약 조건이 피드백을 통해서만 드러나는 방식이 실제 환경의 복잡도를 충분히 반영하는지에 대한 논의 필요

### 📝 초록 기반 상세 설명
실제 환경에서의 계획 수립은 초기에는 명시되지 않았던 환경적, 사용자적 제약 조건이 상호작용을 통해 점진적으로 드러나는 특성을 가집니다. 기존 벤치마크는 이러한 이중 제약 조건 하에서의 적응형 재계획(re-planning) 능력을 충분히 검증하지 못하고 있습니다. 이를 해결하기 위해 307개의 가사 작업 기반 동적 인터랙티브 벤치마크인 AdaPlanBench를 도입합니다. 에이전트는 계획이 제약 조건을 위반할 때만 숨겨진 제약이 드러나는 환경에서 피드백을 통해 제약 조건을 추론하고 계획을 수정해야 합니다. 실험 결과, 최상위 모델조차 67.75%의 정확도에 머물며 제약 조건이 누적될수록 성능이 저하되는 양상을 보였습니다. 특히 사용자 제약 조건은 물리적 근거 부족과 낮은 효율성으로 인해 에이전트에게 큰 도전 과제가 됨을 확인했습니다.

- 기존 External Regret 기반 에이전트와 RP-Regret 기반 에이전트의 협력 효율성 비교

### ⚠️ 리스크/한계

- 상대방의 전략 변화가 매우 빠를 경우 알고리즘의 성능 저하 가능성

RP-Regret은 비볼록(non-convex) 최적화 문제를 포함하지만, 이를 해결하기 위해 최적화 오라클 기반 알고리즘, 선형화 대리 함수 알고리즘, 느린 전략 변화 대응 알고리즘 등 세 가지 접근법을 제안합니다. 연구 결과, 모든 플레이어가 이 지표를 최소화할 때 특정 하위게임 완전 균형(Subgame Perfect Equilibria)을 학습할 수 있음을 확인했습니다. 실험을 통해 Stag-Hunt 게임과 같은 상황에서 더 높은 효용을 주는 협력적 솔루션을 도출할 수 있음을 입증했습니다.

### 📝 초록 기반 상세 설명
최근 LLM이 코딩 에이전트로 활용되면서 안전성 이슈가 단일 응답을 넘어 일련의 작업 시퀀스로 확장되고 있습니다. 기존 벤치마크는 단순 거부 여부에만 집중하여, 상태가 변화하는 워크스페이스 환경에서의 안전성 영향을 제대로 평가하지 못하는 한계가 있습니다. 이를 해결하기 위해 실제 프로젝트 환경과 유사한 상태 유지형 워크스페이스에서 에이전트의 최종 상태를 평가하는 SABER 벤치마크를 제안합니다. SABER는 위반 원인을 분류하여 모델별 안전 프로필 분석을 가능하게 합니다. 실험 결과, 최고 성능 모델조차 54% 이상의 높은 유해 안전 위반율(HSR)을 보여 현재의 정렬 기술이 실무 환경에 불충분함을 시사합니다.
