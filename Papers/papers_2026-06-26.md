## 2. [DomainShuttle: Freeform Open Domain Subject-driven Text-to-video Generation](https://huggingface.co/papers/2606.26058)
**Upvotes**: 55 | **도입 난이도**: 상 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.26058

**태그**: Video-Generation, Diffusion-Models, Personalization, Computer-Vision, Vision, Video

![Thumbnail](images/DomainShuttle_Freeform_Open_Domain_Subject-driven_Text-to-video_Generation_img.jpg)

### 📌 한 줄 요약
In-domain의 높은 재현율과 Cross-domain의 유연성을 동시에 확보한 오픈 도메인 대상 기반 텍스트-비디오 생성 기술

### 🔑 핵심 포인트
- Domain-MoT: 비디오와 참조 특징을 분리하고 도메인 인식 AdaLN을 통해 정교한 모델링 수행
- Video-Reference DualRoPE: 참조 이미지와 비디오 토큰을 별도의 RoPE 공간에 배치하여 정밀한 공간 모델링 구현
- Cross-Pair Consistent Loss: 대상의 본질적 특징만 추출하여 도메인 변화에도 일관된 정체성 유지

### 🧑‍💻 개발자 관점
특정 캐릭터나 사물을 유지하면서도 배경, 스타일, 환경을 자유롭게 바꾸는 고품질 비디오 생성 파이프라인 구축이 가능해집니다.

### 🚀 실무 적용 아이디어
- 제안된 DualRoPE 구조가 기존 DiT(Diffusion Transformer) 구조와 어떻게 호환되는지 분석
- 다양한 스타일(애니메이션, 실사 등)로의 크로스 도메인 전환 성능 테스트
- 복잡한 배경 속에서도 대상의 특징이 왜곡되지 않는지 정성적 평가 수행

### ⚠️ 리스크/한계
- 도메인 전환 시 대상의 미세한 디테일이 손실될 가능성
- 복잡한 공간 모델링(DualRoPE) 도입에 따른 연산 비용 증가

### 📝 초록 기반 상세 설명
최근 오픈 도메인 대상 기반 텍스트-비디오(S2V) 생성 기술이 주목받고 있으나, 기존 방식은 대상의 특징을 유지하는 데만 집중하여 스타일 변경이나 의미적 결합 같은 크로스 도메인 시나리오에서의 유연성이 부족했습니다. 본 연구에서는 대상의 정체성을 유지하면서도 도메인 간 전환이 자유로운 DomainShuttle을 제안합니다. 이를 위해 비디오와 참조 특징을 분리하고 도메인별 모델링을 지원하는 Domain-MoT와, 참조 이미지와 비디오 토큰을 분리된 공간에서 처리하는 Video-Reference DualRoPE를 도입했습니다. 또한, 불필요한 특징에 영향을 받지 않는 핵심 특징 추출을 위해 Cross-Pair Consistent Loss를 적용했습니다. 실험 결과, 제안 방식은 높은 대상 재현율과 생성 유연성을 동시에 달성하며 기존 모델 대비 뛰어난 성능을 입증했습니다.

- ShutterMuse의 추론 속도를 모바일 온디바이스 환경에 맞게 최적화하는 실험 수행

LLM의 단계별 사고 방식인 Chain-of-Thought(CoT)가 멀티모달 작업에서 어떻게 작동하는지 분석하고자 합니다. 연구진은 12개의 다양한 태스크와 22개의 모델을 통해 CoT의 효과와 한계를 체계적으로 평가했습니다. 분석 결과, CoT는 수학적·과학적 추론에는 효과적이지만 시각적 접지(Grounding)나 객체 수 세기와 같은 인지 작업에서는 오히려 성능을 떨어뜨리는 부작용을 보였습니다. 특히 모델이 언어적 성찰은 유지하면서도 시각적 성찰은 점차 감소시키는 'Look Light, Think Heavy' 패턴이 발견되었습니다. 이는 현재의 멀티모달 CoT가 심도 있는 시각적 분석 능력이 부족함을 시사합니다.

최근 감성 TTS 및 음성 변환 기술이 발전하며 비언어적 발성(NVV) 생성이 늘고 있으나, 기존 화자 인증(SV) 시스템은 NVV에서 정체성 유지 능력이 떨어지는 문제가 있습니다. 또한 NVV 데이터로 미세 조정 시 기존 음성 성능이 저하되는 치명적 망각 현상이 발생합니다. 본 논문은 10가지 NVV 유형에 대한 체계적 연구를 바탕으로, Frozen Data2Vec 특징과 ECAPA-TDNN을 결합한 MoE 기반 프레임워크를 제안합니다. 도메인 인식 라우팅을 갖춘 MoE 모듈과 사전 학습된 교사 모델을 활용한 조건부 지식 증류(Conditional Distillation)를 통해 음성 성능을 유지하면서도 NVV 도메인 간 격차를 줄였습니다. 실험 결과, 기존 베이스라인 대비 음성-NVV 간 등가 오류율(EER)을 크게 낮추었으며 음성 성능 또한 향상시켰습니다.

## 30. [Forecasting Future Behavior as a Learning Task](https://huggingface.co/papers/2606.11445)
**Upvotes**: 0 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.11445

**태그**: LLM, Behavior Forecasting, Model Interpretability, Fine-tuning, Reasoning, Evaluation, Inference

![Thumbnail](images/Forecasting_Future_Behavior_as_a_Learning_Task_img.jpg)

### 📌 한 줄 요약
LLM의 추론 과정을 학습하여 미래 행동(재현성, 입력 변화에 따른 반응)을 예측하는 전용 'Behavior Forecaster' 모델 제안

### 🔑 핵심 포인트
- 설명(Explanation) 대신 추론 궤적(Trajectory) 자체를 학습하여 미래 행동을 예측하는 새로운 패러다임 제시
- 인간의 주석 없이 LRM의 출력값만으로 학습 데이터를 구축하여 확장성 확보
- 기존 거대 모델이 텍스트를 읽는 것보다 훨씬 적은 비용으로 더 높은 예측 정확도 달성

### 🧑‍💻 개발자 관점
LLM의 답변 신뢰도를 평가하거나 에이전트의 행동을 예측할 때, 복잡한 설명 생성 없이 궤적 데이터만으로 효율적인 모니터링 및 검증 시스템을 구축할 수 있습니다.

### 🚀 실무 적용 아이디어
- 특정 도메인(예: 코딩)에 특화된 LRM의 추론 로그를 수집하여 소형 예측 모델 학습 테스트
- 학습된 Forecaster를 활용하여 LLM 답변의 일관성(Consistency) 점수 산출 실험
- Backbone 모델 초기화 여부에 따른 예측 성능 차이 검증

### ⚠️ 리스크/한계
- 학습 데이터가 타겟 LRM의 출력에 의존하므로, 모델 업데이트 시 재학습 필요성 존재
- 추론 궤적에 포함되지 않은 외부 환경 변화에 대한 예측력은 제한적일 수 있음

### 📝 초록 기반 상세 설명
기존에는 LLM의 동작 원리를 설명(Explanation)하여 미래 행동을 예측하려 했으나, 긴 추론 궤적(Trajectory)을 자연어로 해석하는 것은 정확도가 낮고 복잡한 문제가 있습니다. 본 논문은 설명 단계를 생략하고 추론 궤적 자체를 학습 데이터로 사용하여 미래 행동을 예측하는 'Behavior Forecaster'를 제안합니다. 별도의 인간 주석 없이 LRM의 출력값만으로 학습 데이터를 생성하며, 추론은 단일 패스로 이루어집니다. 실험 결과, 제안된 방식은 답변 재현성 및 입력 변화에 따른 반응 예측에서 GPT-4나 Claude와 같은 모델이 텍스트를 읽는 것보다 더 높은 정확도를 보였습니다. 특히 타겟 LRM으로부터 초기화하고 엔드투엔드 미세 조정을 수행할 때 가장 강력한 성능을 나타냈습니다.
