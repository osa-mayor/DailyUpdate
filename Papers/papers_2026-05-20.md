**태그**: AI Agents, Software Engineering, LLM, Automation, Agent, Reasoning, Multimodal, Evaluation, Safety

![Thumbnail](images/Code_as_Agent_Harness_img.jpg)

### 📌 한 줄 요약
코드를 단순한 결과물이 아닌 에이전트의 추론, 실행, 검증을 위한 핵심 인프라(Harness)로 정의하는 새로운 프레임워크 제안

### 🔑 핵심 포인트
- 코드를 에이전트의 추론, 행동, 환경 모델링을 연결하는 'Harness Interface'로 정의
- 계획, 메모리, 도구 사용 및 피드백 기반 제어를 포함하는 'Harness Mechanism' 체계화
- 공유 코드 아티팩트를 통한 멀티 에이전트 간의 협업, 리뷰 및 검증 구조 제시

### 🧑‍💻 개발자 관점
에이전트 개발 시 코드를 단순 스크립트가 아닌, 상태 관리와 검증이 가능한 시스템 인프라로 설계해야 함을 시사합니다.

### 🚀 실무 적용 아이디어
- LLM 기반 에이전트의 작업 수행 과정을 코드로 기록하고 이를 피드백 루프에 활용하는 실험
- 멀티 에이전트 환경에서 공유 코드 저장소를 통한 상태 동기화 구조 설계
- 에이전트의 실행 결과에 대한 자동화된 단위 테스트 및 검증 로직 구축

### ⚠️ 리스크/한계
- 코드 실행 시 발생할 수 있는 보안 및 안전성 문제 (Human-in-the-loop 필요)
- 불완전한 피드백 환경에서의 에이전트 제어 및 회귀(Regression) 문제

### 📝 초록 기반 상세 설명
최근 LLM은 코드 생성 능력을 넘어 에이전트 시스템의 핵심 요소로 진화하고 있습니다. 기존에는 코드가 최종 출력물이었으나, 이제는 에이전트의 추론과 환경 모델링을 위한 운영 기질(Substrate) 역할을 수행합니다. 본 논문은 이를 'Code as Agent Harness'라는 통합된 관점으로 정의하고 세 가지 계층(인터페이스, 메커니즘, 스케일링)으로 체계화합니다. 이를 통해 단일 에이전트부터 멀티 에이전트 환경까지 아우르는 코드 중심의 에이전트 인프라 구조를 제시합니다. 결과적으로 실행 가능하고 검증 가능한 AI 에이전트 시스템 구축을 위한 로드맵을 제공합니다.

- 실행 결과의 귀속 분석(Credit Assignment)을 통한 검증된 스킬의 선별적 업데이트 메커니즘

### 🧑‍💻 개발자 관점

### 🚀 실무 적용 아이디어

- 에이전트가 작업 수행 전 관련 스킬을 검색하여 컨텍스트로 사용하는 RAG 방식 적용
- 성공적인 작업 경로를 스킬 라이브러리에 반영했을 때의 성능 향상 폭 측정

- 환경 변화에 민감한 스킬의 경우, 잘못된 업데이트가 전체 시스템 성능을 저하시킬 위험

LLM 에이전트가 생성하는 실행 이력은 재사용 가능한 경험이 될 수 있지만, 노이즈가 많고 관리가 어렵다는 문제가 있습니다. 본 논문은 실행 가능한 스크립트와 절차적 가이드를 결합한 '에이전트 스킬' 개념을 제안합니다. 이를 위해 제안된 SkillsVote 프레임워크는 스킬의 수집, 추천, 진화 과정을 포괄하는 라이프사이클 거버넌스를 제공합니다. 구체적으로 환경 요구사항과 검증 가능성을 바탕으로 스킬을 프로파일링하고, 실행 전 검색과 실행 후 결과 분석을 통해 성공적인 스킬만을 선별하여 업데이트합니다. 실험 결과, 외부 스킬 라이브러리 관리를 통해 모델 업데이트 없이도 벤치마크 성능을 유의미하게 향상시켰습니다.

## 3. [LongLive-2.0: An NVFP4 Parallel Infrastructure for Long Video Generation](https://huggingface.co/papers/2605.18739)
**Upvotes**: 87 | **도입 난이도**: 상 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2605.18739

**태그**: Video Generation, Parallelism, Quantization, Deep Learning Infrastructure, Video, Benchmark, Inference, Distillation

![Thumbnail](images/LongLive-2.0_An_NVFP4_Parallel_Infrastructure_for_Long_Video_Generation_img.jpg)

### 📌 한 줄 요약
NVFP4 정밀도를 활용하여 긴 영상 생성의 학습 및 추론 병목을 해결한 고성능 병렬 인프라 기술

### 🔑 핵심 포인트
- Balanced SP: 효율적인 Teacher-forcing과 시퀀스 병렬 처리를 결합한 AR 학습 기법
- NVFP4 기반 최적화: 학습 시 GEMM 연산 가속 및 추론 시 KV 캐시 양자화를 통한 메모리 절감
- 고성능 추론 파이프라인: 비동기 스트리밍 VAE 디코딩 및 LoRA를 활용한 실시간 생성 지원

### 🧑‍💻 개발자 관점
대규모 비디오 생성 모델의 학습 비용을 절감하고, 실시간에 가까운 고해상도 영상 생성 인프라를 구축하는 데 실질적인 가이드를 제공합니다.

### 🚀 실무 적용 아이디어
- Blackwell GPU 환경에서 NVFP4 양자화 적용 시 성능 변화 측정
- 제안된 Balanced SP 기법을 기존 AR 기반 비디오 모델에 이식 실험
- LoRA 가중치를 이용한 4-step 디노이징 실시간 생성 성능 검증

### ⚠️ 리스크/한계
- Blackwell GPU 아키텍처에 특화된 최적화로 인해 구형 하드웨어에서의 성능 이점 제한적
- 정밀도 저하(W4A4)에 따른 비디오 품질 저하 가능성

### 📝 초록 기반 상세 설명
긴 영상 생성 모델의 학습과 추론 과정에서 발생하는 속도 및 메모리 병목 현상이 주요 문제로 제기되었습니다. 이를 해결하기 위해 NVFP4 정밀도를 기반으로 학습과 추론 전 과정을 최적화한 LongLive-2.0 인프라를 제안합니다. 학습 단계에서는 Balanced SP(Sequence Parallel) 기법을 통해 효율적인 Teacher-forcing 레이아웃을 구현하고, 추론 단계에서는 Blackwell GPU에 최적화된 W4A4 양자화 및 비동기 스트리밍 VAE 디코딩을 적용했습니다. 결과적으로 기존 방식 대비 학습 속도는 최대 2.15배, 추론 속도는 1.84배 향상되었습니다. 특히 5B 모델 기준 45.7 FPS의 높은 추론 성능을 달성하며 고품질의 긴 영상 생성을 가능하게 했습니다.

로봇 제어 시 매 순간 새로운 시각 정보를 처리하는 무거운 모델과 별개로, 가벼운 액션 생성 모듈을 분리하여 실시간성과 동작의 부드러움을 동시에 확보할 수 있는 아키텍처를 제시합니다.

### 🚀 실무 적용 아이디어
- 기존의 Chunk-based action head와 AR-VLA의 궤적 부드러움 비교 실험
- 시각 정보 업데이트 주기가 느린 환경에서의 재앵커링 효과 검증
- 사전 학습된 VLA 백본에 AR 액션 전문가 모듈을 결합하는 파이프라인 구축

### ⚠️ 리스크/한계
- 자기회귀 방식 특유의 오차 누적(Error Accumulation) 가능성
- 장기 메모리 관리에 따른 연산 복잡도 증가

### 📝 초록 기반 상세 설명
기존의 VLA 모델과 확산 정책(Diffusion Policy)은 매 관측마다 시간적 맥락을 초기화하여 동작의 연속성이 떨어지는 문제가 있었습니다. 이를 해결하기 위해 본 논문은 시각-언어 접두사(Prefix)를 조건으로 하되, 자체적인 장기 메모리를 통해 동작 이력을 유지하는 독립적인 자기회귀(AR) 액션 전문가를 제안합니다. 이 구조는 빠른 제어와 느린 추론 사이의 주파수 불일치 문제를 해결하며, 인지 지연을 수학적으로 처리하는 재앵커링(Re-anchoring) 메커니즘을 통해 비동기적 모달리티를 동기화합니다. 실험 결과, 제안된 방식은 기존의 청크 기반 액션 헤드를 대체할 수 있으며, 더 매끄러운 궤적과 높은 성공률을 보여주었습니다. 결과적으로 이 모델은 시공간적으로 일관된 동작 생성을 가능하게 하는 확장 가능한 구조를 제공합니다.
