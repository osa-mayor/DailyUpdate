### 🔑 핵심 포인트
- 실행 환경(Docker 등) 없이 코드 패치의 정답 여부를 판단하는 환경 독립적 검증 방식 제안
- 에이전트의 저장소 탐색(Exploration)을 통해 수집된 증거를 기반으로 한 지능적 검증 메커니즘
- SFT 데이터 필터링 및 RL 보상 모델로 활용 가능한 효율적인 포스트 트레이닝 파이프라인 구축

### 🧑‍💻 개발자 관점
코드 실행 환경 구축에 드는 막대한 컴퓨팅 자원과 시간을 절약하면서도, 높은 수준의 코드 검증 및 모델 학습이 가능해집니다.

### 🚀 실무 적용 아이디어
- 에이전트 기반 코드 탐색 로직이 실제 복잡한 의존성을 가진 프로젝트에서 유효한지 테스트
- 실행 기반 검증기와 Dockerless 간의 검증 일치율(Agreement rate) 비교 실험
- SFT 데이터셋 정제 시 Dockerless를 필터로 적용하여 데이터 품질 변화 관찰

### ⚠️ 리스크/한계
- 코드 실행 없이 논리적 추론만으로 검증하므로, 런타임 에러나 복잡한 사이드 이펙트를 놓칠 가능성 존재
- 에이전트의 탐색 능력이 부족할 경우 잘못된 증거를 바탕으로 오판할 위험

### 📝 초록 기반 상세 설명
코딩 에이전트 학습을 위한 프로그램 검증은 일반적으로 Docker와 같은 격리된 실행 환경을 필요로 하며, 이는 막대한 설정 비용을 발생시킵니다. 본 논문은 코드를 직접 실행하지 않고도 패치의 정답 여부를 판단하는 'Dockerless'를 제안합니다. Dockerless는 단순히 참조 코드와 비교하는 대신, 에이전트가 저장소를 탐색하며 수집한 증거를 바탕으로 패치의 정확성을 판단합니다. 벤치마크 결과, Dockerless는 기존 오픈소스 검증기보다 14.3 AUC만큼 높은 성능을 보였습니다. 이를 SFT 필터 및 RL 보상 모델로 활용한 결과, 환경 구축 없이도 기존 실행 기반 방식에 필적하는 높은 SWE-bench 해결 능력을 달리기했습니다.

## 3. [DOPD: Dual On-policy Distillation](https://huggingface.co/papers/2606.30626)
**Upvotes**: 75 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.30626

**태그**: Knowledge Distillation, LLM, VLM, On-policy Learning, Vision, Distillation

![Thumbnail](images/DOPD_Dual_On-policy_Distillation_img.jpg)

### 📌 한 줄 요약
특권 정보로 인한 성능 왜곡(Privilege Illusion)을 방지하기 위해 교사와 학생 간의 이득 차이를 고려하여 토큰별로 감독 신호를 동적으로 배분하는 이중 증류 기법

### 🔑 핵심 포인트
- Privilege Illusion: 특권 정보 주입 시 발생하는 능력 차이와 정보 비대칭의 혼동 문제 정의
- DOPD: Advantage-aware dual distillation을 통한 토큰별 동적 감독 라우팅 메커니즘
- Token-level Granularity: 토큰별 중요도와 정책 차이를 고려한 적응형 지식 전이

### 🧑‍💻 개발자 관점
모델 경량화나 지식 증류 시, 외부 데이터나 추가 정보를 활용할 때 발생하는 성능 왜곡을 막는 정교한 학습 전략을 제공합니다.

### 🚀 실무 적용 아이디어
- 특권 정보(예: Ground Truth, Chain-of-Thought)를 포함한 데이터셋에서 기존 OPD와 DOPD의 성능 비교 실험
- 토큰별 중요도(Advantage) 계산 로직이 실제 학습 안정성에 미치는 영향 분석
- LLM과 VLM 환경에서의 범용성 검증

### ⚠️ 리스크/한계
- 토큰 단위의 동적 라우팅으로 인해 학습 과정의 복잡도 및 연산 비용 증가 가능성
- Advantage 계산 방식에 따른 하이퍼파라미터 민감도 존재 가능성

### 📝 초록 기반 상세 설명
On-policy distillation(OPD)은 학생 모델이 생성한 궤적에 밀집된 토큰 단위 신호를 제공하여 우수한 지식 전이를 가능하게 합니다. 하지만 교사나 학생에게 특권 정보(privileged information)를 주입할 경우, 학생이 학습해야 할 능력 차이와 단순히 모방만 가능한 정보 비대칭을 혼동하는 'Privilege Illusion' 문제가 발생합니다. 또한 모든 토큰이 동일한 중요도를 갖지 않는 비균일성 문제도 존재합니다. 이를 해결하기 위해 본 논문은 Advantage Gap과 상대적 확률에 따라 교사와 학생 중 누구로부터 감독을 받을지 동적으로 결정하는 DOPD 프레임워크를 제안합니다. 각 토큰은 상황에 따라 서로 다른 강도와 전략의 감독을 받으며, 이를 통해 신뢰할 수 있는 능력 전이와 보조 신호 획득을 동시에 달성합니다. 실험 결과 LLM 및 VLM 환경에서 기존 OPD 대비 뛰어난 성능과 안정성, 강건성을 입증하였습니다.

### 📌 한 줄 요약
입력 데이터의 특성에 맞춰 최적의 디퓨전 블록 크기를 동적으로 결정하여 추론 효율을 극대화하는 적응형 스펙큘레이티브 디코딩 기술

### 🔑 핵심 포인트
- 고정된 블록 크기 대신 입력 샘플별로 최적의 블록 크기를 결정하는 인스턴스 적응형 정책 제안
- 프리필링 단계의 표현(representation)을 활용하여 단 한 번의 예측으로 블록 크기를 결정하는 경량화된 메커니즘

### 🧑‍💻 개발자 관점

### 🚀 실무 적용 아이디어
- 다양한 모델 크기(예: 7B, 14B)에서 블록 크기 예측 정확도와 추론 속도 간의 트레이드오프 측정
- 프리필링 단계의 임베딩이 블록 크기 결정에 미치는 영향력 분석
- 기존 고정 블록 방식 대비 실제 서비스 환경에서의 지연 시간(Latency) 개선 효과 검증

### ⚠️ 리스크/한계
- 프리필링 단계의 연산 오버헤드가 전체 추론 속도에 미치는 영향
- 학습된 블록 크기 예측 정책이 학습 데이터와 완전히 다른 도메인의 텍스트에서 범용성을 유지할 수 있는지 여부

### 📝 초록 기반 상세 설명
최근 디퓨전 기반 스펙큘레이티브 디코딩은 블록 단위 생성을 통해 높은 병렬성을 제공하며 SOTA 성능을 달성했습니다. 그러나 기존 방식은 고정된 블록 크기를 사용하여 모든 입력에 대해 동일한 전략을 적용한다는 한계가 있습니다. 본 논문은 최적의 블록 크기가 샘플마다 다르며, 특정 구조를 가진다는 점을 발견했습니다. 이를 해결하기 위해 프리필링(prefilling) 단계의 표현을 기반으로 최적 블록 크기를 예측하는 'BlockPilot' 정책 학습 모델을 제안합니다. 이 방식은 단 한 번의 예측으로 동작하며 기존 모델에 플러그앤플레이 방식으로 통합 가능합니다. 실험 결과, 최소한의 오버헤드로 Qwen3-4B 모델에서 유의미한 속도 향상과 수락 길이를 달성했습니다.

- 고품질 모드와 저지연 모드 간의 유연한 스위칭 가능

### 🧑‍💻 개발자 관점
실시간 음성 인터랙션 서비스 개발 시, 상황에 따라 응답 속도와 음성 품질을 실시간으로 최적화할 수 있는 기술적 토대를 제공합니다.

### 🚀 실무 적용 아이디어
- 제공된 오디오 샘플을 통해 프레임 레이트 변화에 따른 음성 품질 저하 정도 확인
- 다양한 네트워크 환경을 가정하여 프레임 레이트 조절에 따른 추론 속도 변화 측정
- 기존 고정 프레임 레이트 모델과의 품질/속도 트레이드오프 곡선 비교

### ⚠️ 리스크/한계
- 프레임 레이트를 극단적으로 낮출 경우 음성 품질 및 자연스러움 저하 가능성
- 동적 프레임 레이트 처리를 위한 복잡한 토크나이저 및 모델 구조 필요

### 📝 초록 기반 상세 설명
기존의 음성 언어 모델(SLM)은 고정된 프레임 레이트를 사용하여 음성 데이터의 가변적인 정보 밀도를 반영하지 못하며, 추론 시 품질과 속도 간의 트레이드오프가 불가능했습니다. 본 논문에서는 동적 프레임 레이트 표현 방식을 SLM에 최초로 적용한 FlexiSLM을 제안합니다. 이 모델은 입력과 출력 모두에서 프레임 레이트를 동적으로 조절할 수 있는 기능을 제공합니다. 실험 결과, 고품질 모드에서 기존 7B급 모델들을 능가하는 성능을 보였습니다. 또한, 프레임 레이트를 낮춤으로써 품질을 유지하면서도 추론 시간을 획기적으로 단축할 수 있음을 입증했습니다.

## 35. [SWE-INTERACT: Reimagining SWE Benchmarks as User-Driven Long-Horizon Coding Sessions](https://huggingface.co/papers/2606.30573)
**Upvotes**: 0 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.30573

**태그**: AI Agent, Software Engineering, Benchmark, Human-in-the-loop, Agent, Vision, Evaluation

![Thumbnail](images/SWE-INTERACT_Reimagining_SWE_Benchmarks_as_User-Driven_Long-Horizon_Coding_Sessions_img.jpg)

### 📌 한 줄 요약
단일 작업 해결을 넘어, 모호한 요구사항과 피드백이 오가는 실제 협업 환경에서의 AI 에이전트 성능을 평가하는 새로운 벤치마크 제안

### 🔑 핵심 포인트
- 사용자 시뮬레이터를 통한 모호하고 진화하는 요구사항 환경 구축
- 단일 작업(Single-turn) 성능과 다회차 상호작용(Multi-turn) 성능 간의 괴리 입증
- 에이전트의 목표 발견(Goal Discovery) 및 반복적 정교화(Iterative Refinement) 능력 평가

### 🧑‍💻 개발자 관점
실제 개발 환경에서는 요구사항이 불완전하고 대화로 구체화되므로, 단순 코드 생성 능력을 넘어 협업 능력을 갖춘 에이전트 개발의 중요성을 시사합니다.

### 🚀 실무 적용 아이디어
- 현재 사용 중인 코딩 에이전트에 모호한 프롬프트를 주고 단계적 피드백을 주는 시나리오 테스트
- 에이전트가 이전 대화의 제약 조건을 유지하며 코드를 수정하는지 확인하는 워크플로우 설계
- 에이전트의 '과도한 자율성(Over-agentic coding)'이 코드 품질에 미치는 영향 분석

### ⚠️ 리스크/한계
- 사용자 시뮬레이터의 설계 방식에 따라 평가 결과가 편향될 가능성
- 실제 인간 개발자의 복잡한 의도와 감정적 맥락을 완벽히 재현하기 어려움

### 📝 초록 기반 상세 설명
기존의 SWE 벤치마크는 모든 요구사항이 사전에 정의된 상태에서 에이전트의 자율적 구현 능력만을 평가하는 한계가 있었습니다. 이를 해결하기 위해 연구진은 사용자 시뮬레이터가 모호한 지시를 내리고, 피드백과 제약 조건을 점진적으로 추가하는 상호작용형 벤치마크인 SWE-Interact를 도입했습니다. 실험 결과, 단일 작업에서 뛰어난 모델이 다회차 상호작용 환경에서는 성능이 급격히 저하되는 현상이 관찰되었습니다. 최상위 모델들은 모호함 속에서도 목표를 찾아내고 코드를 통합하는 능력을 보였으나, 여전히 과도한 자율성이나 요구사항 망각 문제를 겪었습니다. 결과적으로 SWE-Interact는 에이전트의 목표 발견 및 반복적 정교화 능력을 측정하는 새로운 평가 축을 제시합니다.
