### 🚀 실무 적용 아이디어

- 유튜브 튜토리얼 등 영상 데이터를 스텝별 실행 코드로 변환하는 파이프라인 구축 테스트
- 계층적 스킬 구조가 에이전트의 복합 작업 수행 능력에 미치는 영향 분석

### ⚠️ 리스크/한계
- 멀티모달 데이터(특히 영상)에서 정확한 실행 코드를 추출할 때 발생하는 노이즈 및 오류 가능성

### 📝 초록 기반 상세 설명

### 📌 한 줄 요약
추출과 정제 단계를 분리하고 경량 모델을 최적화하여 성능과 효율을 동시에 잡은 모듈형 GraphRAG 엔진

### 🔑 핵심 포인트
- 추출(Extraction)과 통합(Consolidation) 단계를 분리하여 데이터 품질과 검색 신뢰도 확보
- 언어 능력 중심의 경량 모델(Meno-Lite-0.1) 학습을 통한 효율적인 지식 그래프 구축

### 🧑‍💻 개발자 관점
단일 GPU 환경에서도 고성능 지식 그래프를 구축할 수 있어, 리소스 제약이 있는 실무 환경에서 GraphRAG를 도입하기 용이합니다.

### 🚀 실무 적용 아이디어
- pip install graph_ragu를 통한 로컬 환경 설치 및 기본 워크플로우 테스트
- 도메인 특화 데이터셋을 활용한 Meno-Lite-0.1의 추출 성능 검증
- 기존 RAG 시스템과 RAGU의 검색 결과(Evidence Recall) 비교 실험

### ⚠️ 리스크/한계
- 특정 도메인에 최적화된 모델이므로 일반적인 범용 지식 구축 시 성능 변화 가능성
- 복잡한 그래프 구조 생성 시 연산 비용이 발생할 수 있음

### 📝 초록 기반 상세 설명
기존 GraphRAG 시스템은 단일 추출 과정으로 인해 노이즈가 섞인 엔티티와 취약한 검색 성능 문제를 겪고 있었습니다. RAGU는 추출과 통합 과정을 분리하여 2단계 타입 추출, DBSCAN 기반 중복 제거, Leiden 커뮤니티 탐지 과정을 거치는 모듈형 엔진을 제안합니다. 특히 추출에 필요한 언어 능력은 모델 크기에 비례하지 않는다는 점에 착안하여, 7B 규모의 Meno-Lite-0.1 모델을 학습시켰습니다. 실험 결과, Meno-Lite-0.1은 더 큰 모델인 Qwen2.5-32B보다 지식 그래프 구축 성능이 우수하며 GraphRAG-Bench에서도 높은 재현율을 기록했습니다. 이 엔진은 단일 GPU에서 구동 가능하며 오픈소스로 공개되었습니다.

카메라 시점이 아닌 로봇 중심의 3D 좌표 정보를 이미지 형태로 제공하여 VLA 모델의 시점 변화에 대한 일반화 성능을 높이는 기술

### 🔑 핵심 포인트
- 카메라 시점과 액션 정의 시점 간의 프레임 불일치 문제 해결
- 기존 2D VLA 구조를 유지하며 3D 기하 정보를 주입하는 Pointmap 방식 제안
- 학습 시와 다른 카메라 위치에서도 동작하는 시점 일반화 성능 확보

### 🧑‍💻 개발자 관점
로봇 제어 모델을 설계할 때, 카메라 위치가 바뀔 때마다 모델을 재학습하는 대신 입력 데이터의 좌표계를 정렬하여 일반화 성능을 높이는 실무적 전략을 제시합니다.

### 🚀 실무 적용 아이디어
- 기존 RGB 입력 대신 3D 좌표가 포함된 Pointmap을 입력으로 넣어 성능 변화 관찰
- 학습 시와 완전히 다른 각도의 카메라를 배치하여 일반화 성능 테스트
- 기존 VLA 모델의 입력 레이어에 Pointmap을 결합하는 프로토타입 구현

### ⚠️ 리스크/한계
- Pointmap 생성을 위한 정밀한 3D 센서 데이터 또는 역기하학 계산 필요
- 카메라와 로봇 사이의 캘리브레이션 오차에 따른 성능 저하 가능성

### 📝 초록 기반 상세 설명
기존의 Vision-Language-Action (VLA) 모델은 카메라 시점의 이미지를 관찰하지만, 로봇의 액션은 로봇 중심의 3D 좌표계에서 정의되어 시점 불일치 문제가 발생합니다. 고정된 시점에서는 문제가 적지만, 다양한 카메라 환경을 포함하는 대규모 데이터셋에서는 시점 변화에 따른 일반화가 어려워집니다. 이를 해결하기 위해 픽셀에 로봇 프레임 기준 3D 좌표를 저장하는 'Robot-centric Pointmaps'를 제안합니다. 이 방식은 기존 2D VLA의 구조를 거의 변경하지 않으면서도 로봇 중심의 기하학적 정보를 제공할 수 있습니다. 실험 결과, RoboCasa 환경에서 기존 베이스라인보다 우수한 성능을 보였으며, 학습 시 보지 못한 새로운 카메라 위치에서도 RGB 전용 정책보다 높은 강건함을 입증했습니다.
