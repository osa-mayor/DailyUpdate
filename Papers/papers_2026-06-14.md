## 2. [MiniMax Sparse Attention](https://huggingface.co/papers/2606.13392)
**Upvotes**: 99 | **도입 난이도**: 중 | **신뢰도**: 상
**arXiv**: https://arxiv.org/abs/2606.13392

**태그**: Long-Context, LLM-Inference, Sparse-Attention, GPU-Optimization, Agent, RAG, Reasoning, Multimodal, Evaluation, Inference

![Thumbnail](images/MiniMax_Sparse_Attention_img.jpg)

### 📌 한 줄 요약
GQA 기반의 블록 단위 희소 어텐션(MSA)을 통해 1M 컨텍스트 환경에서 연산량을 획기적으로 줄이면서도 성능을 유지하는 기술

### 🔑 핵심 포인트
- GQA 구조를 활용한 블록 단위 희소 어텐션(MSA) 설계
- Index Branch를 통한 그룹별 Top-k 블록 선택 및 Main Branch의 정밀 어텐션 결합
- GPU 텐서 코어 활용도를 높이기 위한 exp-free Top-k 및 전용 커널 최적화

### 🧑‍💻 개발자 관점
수십만 토큰 이상의 초장기 컨텍스트를 다루는 에이전트나 코드 분석 워크플로우에서 추론 비용과 지연 시간을 획기적으로 낮출 수 있습니다.

### 🚀 실무 적용 아이디어
- 제공된 GitHub 커널을 사용하여 기존 GQA 모델과 추론 속도 및 메모리 점유율 비교
- 1M 이상의 긴 컨텍스트 환경에서 정보 회수(Retrieval) 성능 테스트
- 다양한 GPU 아키텍처에서의 커널 최적화 효율성 검증

### ⚠️ 리스크/한계
- Top-k 선택 방식에 따른 특정 컨텍스트에서의 정보 손실 가능성
- 블록 단위 접근 방식이 매우 미세한(fine-grained) 어텐션이 필요한 작업에 미치는 영향

### 📝 초록 기반 상세 설명
초장기 컨텍스트 처리가 필수적인 LLM 환경에서 기존 소프트맥스 어텐션의 이차 복잡도 문제는 배포 규모 확장을 어렵게 만듭니다. 이를 해결하기 위해 GQA를 기반으로 한 블록 단위 희소 어텐션인 MiniMax Sparse Attention(MSA)을 제안합니다. Index Branch가 KV 블록을 스코어링하여 그룹별로 Top-k 블록을 선택하면, Main Branch가 선택된 블록에 대해서만 정밀한 어텐션을 수행합니다. 또한 GPU 연산 효율을 극대화하기 위해 exp-free Top-k 선택 및 KV-outer sparse attention이 적용된 전용 커널을 공동 설계했습니다. 실험 결과, 109B 모델에서 1M 컨텍스트 기준 연산량을 28.4배 절감하면서도 GQA 수준의 성능을 유지하며 압도적인 추론 속도 향상을 달성했습니다.
