- 검증 가능한 보상(Verifiable rewards)에서 세밀한 증명 단계 보상(Proof-level RL)으로 이어지는 2단계 RL 파이프라인
- 테스트 타임 스케일링(Test-time scaling)을 통한 장기 추론 성능 극대화

복잡한 논리적 추론이 필요한 에이전트 설계 시, 단순 데이터 증강을 넘어 학습 단계별 보상 설계와 테스트 타임 연산량 조절이 성능에 미치는 영향을 보여줍니다.

### 🚀 실무 적용 아이디어
- SFT 단계에서 모델의 자기 검증(Self-checking) 능력을 유도하는 데이터 커리큘럼 설계 실험
- 검증 가능한 정답(Ground truth)이 있는 도메인에서 2단계 RL 파이프라인 적용 테스트
- 추론 토큰 길이에 따른 성능 변화를 확인하기 위한 테스트 타임 스케일링 실험

### ⚠️ 리스크/한계

- 매우 긴 추론 경로(100K+ 토큰) 사용 시 발생하는 막대한 컴퓨팅 비용

### 📝 초록 기반 상세 설명

## 2. [Causal Forcing++: Scalable Few-Step Autoregressive Diffusion Distillation for Real-Time Interactive Video Generation](https://huggingface.co/papers/2605.15141)

![Thumbnail](images/Causal_Forcing++_Scalable_Few-Step_Autoregressive_Diffusion_Distillation_for_Real-Time_Interactive_V_img.jpg)

### 📌 한 줄 요약
초저지연 실시간 비디오 생성을 위해 1~2단계의 프레임 단위 자기회귀(AR) 디퓨전 증류 기술을 제안함

### 🔑 핵심 포인트
- 프레임 단위의 초저지연(1~2 step) 자기회귀 디퓨전 증류 프레임워크 제안
- Causal CD(Causal Consistency Distillation)를 통한 효율적이고 확장 가능한 초기화 전략
- 전체 PF-ODE 궤적 저장 없이 인접 타임스텝 간의 온라인 감독을 통한 학습 최적화

### 🧑‍💻 개발자 관점
실시간 게임 엔진이나 인터랙티브 AI 에이전트 환경에서 지연 시간 없이 고품질 비디오를 생성해야 하는 엔지니어에게 매우 유용한 최적화 기법입니다.

### 🚀 실무 적용 아이디어
- 제공된 GitHub 코드를 활용하여 프레임 단위 vs 청크 단위 생성 속도 및 품질 비교 실험
- 특정 액션 조건(Action-conditioned)이 주어졌을 때의 월드 모델 생성 일관성 테스트
- 제한된 컴퓨팅 자원 환경에서 학습 비용 절감 효과 검증

### ⚠️ 리스크/한계
- 프레임 단위 생성 시 발생할 수 있는 장기적 시간적 일관성(Temporal Consistency) 유지 문제
- 매우 적은 샘플링 스텝(1-2 step)으로 인한 생성 품질 저하 가능성

### 📝 초록 기반 상세 설명
실시간 상호작용 비디오 생성을 위해서는 낮은 지연 시간과 정교한 제어가 필수적입니다. 기존의 청크 단위(chunk-wise) AR 증류 방식은 응답 해상도가 낮고 샘플링 지연이 발생하는 한계가 있었습니다. 본 논문은 1~2단계의 프레임 단위 자기회귀라는 더 도전적인 환경을 설정하고, 초기화 과정에서의 정렬 문제를 해결하기 위한 Causal Forcing++를 제안합니다. 제안된 방식은 전체 궤적을 미리 계산할 필요 없이 인접 타임스텝 간의 온라인 교사 ODE 단계를 활용하여 효율적인 초기화를 수행합니다. 실험 결과, 기존 방식 대비 성능(VBench, VisionReward)을 높이면서도 첫 프레임 지연 시간을 50% 단축하고 학습 비용을 4배가량 절감했습니다. 또한 이 기술을 액션 조건부 월드 모델 생성으로 확장할 수 있음을 보여주었습니다.

## 3. [Self-Distilled Agentic Reinforcement Learning](https://huggingface.co/papers/2605.15155)

**arXiv**: https://arxiv.org/abs/2605.15155

**태그**: Agent, Reinforcement Learning, Self-Distillation, LLM, RAG, Vision, Evaluation, Distillation

![Thumbnail](images/Self-Distilled_Agentic_Reinforcement_Learning_img.jpg)

### 📌 한 줄 요약
멀티턴 에이전트의 불안정성을 해결하기 위해 토큰 단위의 정교한 가이드를 게이트 방식으로 결합한 강화학습 최적화 기법

### 🔑 핵심 포인트
- RL의 희소한 보상 문제를 해결하기 위한 토큰 단위 밀집 가이드(OPSD) 도입
- 멀티턴 환경의 불안정성을 방지하기 위해 OPSD를 게이트형 보조 목적 함수로 설계
- 잘못된 교사 피드백(Negative Rejection)에 의한 성능 저하를 방지하는 신호 감쇄 메커니즘

### 🧑‍💻 개발자 관점
복잡한 멀티턴 태스크를 수행하는 에이전트를 학습시킬 때, RL의 불안정성을 제어하면서도 정교한 행동 가이드를 학습에 반영할 수 있는 프레임워크를 제공합니다.

### 🚀 실무 적용 아이디어
- GRPO와 같은 최신 RL 알고리즘에 SDAR의 게이트형 보조 손실 함수 적용 실험
- 에이전트의 작업 복잡도(Turn 수) 변화에 따른 게이트 파라미터 민감도 테스트
- 교사 모델(Teacher)과 학생 모델(Student) 간의 스킬 격차에 따른 성능 변화 관찰

### ⚠️ 리스크/한계
- 교사 모델의 품질에 따라 게이트 신호가 편향될 위험 존재
- 멀티턴 환경에서 게이트 메커니즘의 하이퍼파라미터 튜닝 난이도

### 📝 초록 기반 상세 설명
LLM 에이전트 학습에서 RL은 보상이 희소하여 장기 작업 학습에 한계가 있습니다. 기존의 On-Policy Self-Distillation(OPSD)은 밀집된 가이드를 제공하지만, 멀티턴 환경에서는 불안정성이 증폭되고 잘못된 가이드로 인한 성능 저하 문제가 발생합니다. 이를 해결하기 위해 SDAR은 OPSD를 RL의 주 목적 함수가 아닌 게이트형 보조 목적 함수로 재정의합니다. 토큰 단위 신호를 시그모이드 게이트로 매핑하여, 교사 모델이 승인한 유효한 토큰은 강화하고 잘못된 거절은 완화하는 방식입니다. 실험 결과, SDAR은 GRPO 대비 다양한 벤치마크에서 성능을 크게 향상시켰으며 모델 스케일에 관계없이 안정적인 성능을 보였습니다.

### 🔑 핵심 포인트

### 🚀 실무 적용 아이디어

- 메모리 압축 방식(Summarization vs Retrieval)에 따른 시각적 디테일 손실률 비교 실험

### ⚠️ 리스크/한계
- 멀티 세션 추론과 같은 고난도 작업에서는 현재의 두 방식 모두 성능 한계가 명확함
- 메모리 에이전트 방식은 정보 압축 시 시각적 세부 사항이 유실될 위험이 있음

### 📝 초록 기반 상세 설명
대규모 시각-언어 모델(LVLM)이 긴 대화를 처리하기 위해 Long-context 방식과 Memory-augmented 방식이 사용되지만, 멀티모달 증거가 필요한 상황에서의 체계적인 비교 벤치마크는 부재했습니다. 이를 해결하기 위해 5가지 기억 능력과 다양한 컨텍스트 길이를 포함하는 MemLens 벤치마크를 도입했습니다. 실험 결과, Long-context 모델은 컨텍스트가 길어질수록 성능이 저하되는 반면, 메모리 에이전트는 압축 과정에서 시각적 정밀도가 떨어지는 특성을 보였습니다. 특히 멀티 세션 추론 능력은 두 방식 모두에서 매우 낮게 나타났습니다. 연구 결과는 장기 기억을 위해 Long-context 어텐션과 구조화된 멀티모달 검색을 결합한 하이브리드 아키텍처의 필요성을 시사합니다.

로맨스 스캠이나 투자 사기와 같은 대화형 사기는 다회차 대화를 통해 점진적으로 피해자를 조작하는 특징이 있습니다. 기존 연구는 정적인 탐지에 치중되어 있어, 실제 사기가 시간에 따라 어떻게 진전되는지 모델링하는 데 한계가 있었습니다. 본 논문은 사용자 신고 데이터를 기반으로 20개 카테고리, 11,573개의 대화 인스턴스를 포함하는 PreScam 벤치마크를 제안합니다. 이 데이터셋은 사기 킬체인(Kill Chain)에 따른 계층적 구조와 턴 단위의 심리적 행동 주석을 포함합니다. 실험 결과, 현재의 LLM은 단순 유창함은 뛰어나지만 사기 위험의 고조와 조작 과정을 추적하는 능력은 부족함을 확인했습니다.
