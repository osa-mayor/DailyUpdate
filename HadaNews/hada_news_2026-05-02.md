### 📰 원문 핵심 요약
Mozilla는 Chrome의 Prompt API 제안에 반대 입장을 표명했습니다. Prompt API는 브라우저 내 언어 모델을 웹 API로 노출하는 것을 목표로 하지만, Mozilla는 이 API가 모델별 동작에 맞춘 구현을 장려하여 웹 표준의 상호 운용성을 저해할 수 있다고 우려합니다. 개발자가 특정 모델(예: Edge의 Phi-4-mini)에 맞춰 프롬프트와 기능을 조정할 경우, 다른 브라우저나 모델에서 품질 저하가 발생할 수 있다는 것입니다. Mozilla는 Prompt API가 웹 생태계의 파편화를 심화시킬 가능성을 제기하며 신중한 접근을 요구하고 있습니다.

### 📌 왜 중요한가
Chrome의 Prompt API 제안은 브라우저 내 AI 모델 접근성을 높이려 하지만, Mozilla는 모델 종속적인 구현을 유도하여 웹 호환성을 해칠 수 있다고 비판합니다. 이 문제는 웹 표준화 과정에서 상호 운용성을 확보하고 특정 브라우저나 모델에 종속되지 않는 방향으로 나아가야 함을 시사합니다.

### ✅ 지금 확인할 포인트
- 브라우저 내 언어 모델을 웹 API로 노출하는 Prompt API는 일반적인 형태로는 유용할 수 있지만, 모델별 동작에 맞춘 구현을 부추겨 상호 운용성 위험을 키울 수 있습니다.
- 개발자가 특정 모델에 맞춰 프롬프트와 기능을 조정하면 다른 브라우저나 모델에서 품질 저하가 발생할 수 있습니다.
- Mozilla는 Prompt API가 웹 생태계의 파편화를 심화시킬 가능성을 제기하며 신중한 접근을 요구하고 있습니다.
