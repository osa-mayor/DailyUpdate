## 4. [Copy Fail](https://copy.fail/)
**카테고리**: 보안/프라이버시 | **출처**: hacker_news | **통합 점수**: 0.720
**게시 시각**: 2026-04-30T03:13:53+09:00
**HN 토론**: https://news.ycombinator.com/item?id=47952181

### 📰 원문 요약
'Copy Fail'은 race condition이나 kernel offset 없이도 작동하는 Linux의 취약점이다. `authencesn`의 로직 버그를 통해 `AF_ALG`와 `splice()`를 거쳐 page-cache에 4바이트를 쓰는 방식으로, 2017년 이후 배포된 Linux 배포판을 root 권한으로 만들 수 있다. 단일 Python 스크립트로 여러 배포판에서 동일한 exploit이 가능하다.

### 💬 토론 요약
Linux kernel 암호화 코드 담당자는 AF_ALG exploit의 빈번한 발생에 불만을 표했으며, vendor들이 이 취약점을 심각하게 받아들이지 않아 패치가 늦어지는 것에 대한 혼란이 있다는 의견이 제시되었다.

### 📌 종합 요약
2017년부터 배포된 Linux 배포판에서 발견된 'Copy Fail' 취약점이 공개되었다. 이 취약점은 권한 상승(LPE)을 가능하게 하며, vendor들의 미흡한 대응으로 인해 많은 배포판에서 아직 패치되지 않은 상태다.

### 🔑 핵심 포인트
- Copy Fail 취약점은 race condition 없이도 Linux kernel의 권한 상승을 가능하게 하는 심각한 문제다.
- vendor들의 늑장 대응으로 인해 많은 Linux 배포판에서 해당 취약점이 여전히 해결되지 않고 있다.
- AF_ALG를 이용한 exploit이 반복적으로 발생하는 것은 Linux kernel 보안에 대한 지속적인 위협을 시사한다.
